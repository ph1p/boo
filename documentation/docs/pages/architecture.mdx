# Architecture

## Stack

- **Swift** (macOS 13+) with **AppKit** for terminal UI and **SwiftUI** for sidebar/settings
- **GhosttyKit** — Metal GPU-accelerated terminal emulation via C API ([ghostty-org/ghostty](https://github.com/ghostty-org/ghostty))
- **POSIX PTY** — `forkpty()` via C helper (`CPTYHelper/`)

## Project Structure

```
Boo/
  App/              AppDelegate, MainWindowController, WindowStateCoordinator
  Ghostty/          GhosttyRuntime (singleton), GhosttyView (Metal surface)
  Terminal/         TerminalBackend (PTY lifecycle protocol)
  Models/           Workspace, Pane (+ TabState), SplitTree, AppSettings, Theme
  Plugin/           Core framework (protocol, registry, runtime, DSL)
  Plugins/          Built-in plugins (FileTree, Git, ClaudeCode, Docker, Bookmarks, Snippets, etc.)
  Views/            PaneView, StatusBarView, ToolbarView, SettingsWindow
  Services/         TerminalBridge, RemoteExplorer, BooSocketServer, SparkleUpdater
CGhostty/           C module wrapping ghostty.h
CPTYHelper/         C helper for forkpty()
CIronmark/          C module wrapping ironmark (Rust markdown parser)
Vendor/ghostty/     Ghostty source (git submodule)
```

Plugin code is colocated — all files for a plugin live in one directory under `Boo/Plugins/`. Tests mirror this layout under `Tests/BooTests/Plugins/`.

## State Model

### Global — `AppSettings` (singleton)

Theme, font, layout, status bar toggles, and per-plugin settings. Changes broadcast via `NotificationCenter` (`.settingsChanged`).

### Per-Tab — `TabState`

Each `Pane.Tab` owns a `TabState` — the single source of truth for that tab:

```swift
struct TabState {
    var workingDirectory: String
    var remoteSession: RemoteSessionType?
    var remoteWorkingDirectory: String?
    var shellPID: pid_t = 0
    var title: String
    var foregroundProcess: String = ""

    // Plugin UI
    var expandedPluginIDs: Set<String> = []
    var userCollapsedSectionIDs: Set<String> = []
    var sidebarSectionHeights: [String: CGFloat] = [:]
    var sidebarScrollOffsets: [String: CGPoint] = [:]
    var sidebarSectionOrder: [String: [String]] = [:]
    var selectedPluginTabID: String? = nil
}
```

### Data Flow

```
Ghostty OSC event → PaneView delegate → MainWindowController
  → TerminalBridge (heuristics, remote detection)
  → WindowStateCoordinator.syncBridgeToTab() → writes to TabState
  → runPluginCycle() → TerminalContext (frozen snapshot)
  → PluginRuntime (enrich → freeze → react) → sidebar/status bar
```

**Key components:**

- **WindowStateCoordinator** — centralized state manager between bridge, tab model, and plugin registry
- **TerminalBridge** — event parser and heuristic engine; its `BridgeState` is transient, `TabState` is authoritative
- **MainWindowController** — delegates state management to the coordinator

## Hierarchy

```
Workspace → SplitTree → Pane[] → Tab[] → TabState
```

Each split pane owns its own tab bar and terminal sessions.

## Build

```bash
make setup         # First time: clone Ghostty + build GhosttyKit + build Boo
make run           # Build and launch
make test          # Run tests
make clean-ghostty # Rebuild GhosttyKit after updating Vendor/ghostty
```

Development note: `make run` produces a debug app bundle with Sparkle feed keys removed, so local builds cannot check the production update feed. Release-oriented targets (`make app`, `make zip`, `make dmg`, `make dist`) keep the Sparkle configuration and bundle `Sparkle.framework` inside `Contents/Frameworks`.

## State Contracts

### Single Source of Truth

| Data                    | Source of Truth        | Persistence             | Sync Rule                               |
| ----------------------- | ---------------------- | ----------------------- | --------------------------------------- |
| **Working Directory**   | BridgeState (live)     | TabState                | Sync to TabState on tab switch          |
| **Remote Session**      | BridgeState (live)     | NOT persisted           | Re-detected on restore via process tree |
| **Remote CWD**          | BridgeState (live)     | TabState                | Sync to TabState on tab switch          |
| **Foreground Process**  | BridgeState (live)     | TabState                | Sync to TabState on tab switch          |
| **Shell PID**           | TabState               | SessionStore            | Set once on terminal creation           |
| **Terminal Title**      | BridgeState (live)     | TabState                | Sync to TabState on tab switch          |
| **Agent Session ID**    | TabState               | SessionStore            | Updated by plugin detection             |
| **Plugin Expanded IDs** | WindowStateCoordinator | TabState                | Sync on tab switch                      |
| **Section Heights**     | DetailPanelView        | AppSettings.`__sidebar` | Sync on panel switch, app quit          |
| **Section Order**       | WindowStateCoordinator | AppSettings.`__sidebar` | Sync on reorder, app quit               |
| **Scroll Offsets**      | DetailPanelView        | TabState                | Sync on tab switch                      |
| **Active Plugin Tab**   | WindowStateCoordinator | TabState                | Sync on tab switch                      |
| **Theme**               | AppSettings            | UserDefaults + file     | Immediate                               |
| **Sidebar Visible**     | MainWindowController   | NOT persisted           | Runtime only                            |

### Key Invariants

1. `BridgeState` must be synced to `TabState` via `syncBridgeToTab()` before any tab switch.
2. Remote session is intentionally not persisted — always re-detected on restore.
3. Plugin settings use namespace: `pluginSettings[pluginID][key]`.
4. Changes to `AppSettings` emit `.settingsChanged` via `NotificationCenter`.

### Sync Points

**Tab Switch:**

1. `syncBridgeToTab(previousPane, previousTabIndex)` — save bridge → tab
2. `savePluginState()` — save plugin UI → tab
3. `restorePluginState(newTab)` — restore plugin UI ← tab
4. `bridge.restoreTabState(...)` — restore bridge ← tab

**App Quit:**

1. For each workspace: sync bridge + plugin state
2. `saveSidebarHeightsToSettings()`
3. `saveSidebarOrderToSettings()`
4. `SessionStore.save(appState)`

### Controller Responsibilities

| Controller                 | Responsibility                                          |
| -------------------------- | ------------------------------------------------------- |
| **MainWindowController**   | Window lifecycle, menu actions, high-level coordination |
| **SidebarController**      | Panel visibility, tab selection, heights sync           |
| **WindowStateCoordinator** | Bridge↔Tab sync, plugin state save/restore              |
| **TerminalBridge**         | Event parsing, heuristics, remote detection             |
