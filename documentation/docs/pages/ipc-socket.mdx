# IPC Socket

Boo exposes a Unix domain socket at `~/.boo/boo.sock` (also available as `$BOO_SOCK` in terminal sessions) for communication between child processes, plugins, and the app.

## Protocol

Newline-delimited JSON. One request per line, one response per request. Long-lived connections can subscribe to push events.

## Commands

| Category       | Commands                                                                          |
| -------------- | --------------------------------------------------------------------------------- |
| **Query**      | `get_context`, `get_theme`, `get_settings`, `get_workspaces`                      |
| **Control**    | `set_theme`, `toggle_sidebar`, `switch_workspace`, `new_tab`, `new_workspace`, `send_text` |
| **Process**    | `set_status`, `clear_status`, `list_status`                                       |
| **Events**     | `subscribe`, `unsubscribe`                                                        |
| **Status Bar** | `statusbar.set`, `statusbar.clear`, `statusbar.list`                              |
| **Plugin**     | `<namespace>.<action>`                                                            |

## Examples

```bash
# Query current terminal context
echo '{"cmd":"get_context"}' | nc -U "$BOO_SOCK"

# Register as an AI agent
echo '{"cmd":"set_status","pid":'"$$"',"name":"claude","category":"ai"}' | nc -U "$BOO_SOCK"

# Send a command to the active terminal
echo '{"cmd":"send_text","text":"echo hello\n"}' | nc -U "$BOO_SOCK"

# Push a status bar segment
echo '{"cmd":"statusbar.set","id":"ci","text":"CI: passing","icon":"checkmark.circle","tint":"green"}' \
  | nc -U "$BOO_SOCK"
```

## Event Subscriptions

Keep the connection open after `subscribe` to receive push events:

| Event                    | Data                                |
| ------------------------ | ----------------------------------- |
| `cwd_changed`            | `path`, `is_remote`, `pane_id`      |
| `title_changed`          | `title`, `pane_id`                  |
| `process_changed`        | `name`, `category`, `pane_id`       |
| `remote_session_changed` | `type`, `host`, `active`, `pane_id` |
| `focus_changed`          | `pane_id`                           |
| `workspace_switched`     | `workspace_id`                      |
| `theme_changed`          | `name`, `is_dark`                   |
| `settings_changed`       | `topic`                             |

Use `"events": ["*"]` to subscribe to all events.

## `send_text`

Write text directly to the active terminal's PTY — identical to typing it.

```bash
echo '{"cmd":"send_text","text":"cd ~/projects/myapp\n"}' | nc -U "$BOO_SOCK"
```

## Status Bar Segments

External processes can publish status bar segments. Tint accepts named colors (`red`, `green`, `yellow`, `blue`, `orange`, `purple`, `accent`) or `#hex`. Segments are auto-cleaned on disconnect.

## Security

- Socket I/O runs on a dedicated serial queue
- Connecting processes are verified with `LOCAL_PEERCRED`
- Registered PIDs are checked as descendants of the shell
- Dead processes and disconnected clients are cleaned up automatically
