# Features

## Workspaces

Open folders as workspaces with their own file tree using **Cmd+N** (home directory) or **Cmd+Shift+O** (choose folder). Each workspace maintains its own state, color, and name.

- **Workspace colors** — Assign preset or custom colors per workspace
- **Pinning** — Pin frequently used workspaces
- **Renaming** — Double-click a workspace pill to rename it, or use the right-click context menu
- **Session persistence** — Workspace names, colors, split layouts, tab paths, and split pane ratios are saved to `~/.boo/session.json` and restored on next launch. Remote connections are not restored; tabs reopen at their local working directory.
- **Drag to switch workspace** — While dragging a tab, hover over a workspace pill for ~500 ms to spring-load into that workspace, then drop the tab onto any pane there.

## Split Panes

Split your terminal in any direction:

- **Cmd+D** — Split right
- **Cmd+Shift+D** — Split down
- **Cmd+]** / **Cmd+[** — Navigate between panes

Each split pane is independent with its own tab bar and terminal sessions. Drag the divider between panes to resize; the ratio is saved and restored across launches.

## Per-Pane Tabs

Each pane has its own tab bar. Create new tabs with **Cmd+T**, close with **Cmd+W**. Closed tabs can be reopened with **Cmd+Z**.

Smart close order: tab → pane → workspace.

- **Right-click a tab** — Context menu with "Close Tab" (or "Close Workspace" when it's the only tab) and a "Move Tab to Workspace →" submenu listing all other open workspaces.
- **Drag tabs across workspaces** — Drag a tab to a different pane or split zone in the current workspace, or use the workspace pill hover to move it to another workspace entirely. Moving the last tab out of a workspace prompts to close that workspace.

## File Explorer

Toggle the sidebar with **Cmd+B**. The explorer shows a live-updating file tree for the current workspace.

- **Local workspaces** — Uses FSEvents for real-time file system watching
- **Remote SSH / Mosh** — Auto-detects SSH-style sessions and reuses shared connections for file listing when possible
- **Remote containers / VMs / devices** — Auto-detects interactive `docker`, `podman`, `nerdctl`, `kubectl`, `oc`, `lxc`, `limactl`, `colima`, `distrobox`, `toolbox`, `vagrant`, and `adb shell` sessions and runs the appropriate listing command for each tool

**File click behavior:**

- **Source/text files** — Opens a new tab in the file's parent directory and runs the configured editor (set in General Settings → File Editor, falls back to `$EDITOR`, then `vi`)
- **Images** — Rendered inline using the Kitty Graphics Protocol (supported natively by Ghostty); otherwise opens with the default macOS app
- **Other files** — Opens with the default macOS app

Configure which extensions open in the editor (and toggle inline image preview) in **Settings → Plugins → Files**.

## Sidebar Plugins

The sidebar shows stacked, collapsible panels powered by the [plugin system](/plugins):

- **Files** — Live file tree for the current directory. Supports local (FSEvents) and remote (SSH, Mosh, containers, VMs, and device shells) browsing. Click to open, right-click for context menu.
- **Git** — Branch name, staged/unstaged/untracked files, stash count, ahead/behind. Inline stage/unstage buttons. Refresh on file system changes.
- **Docker** — Running containers with exec, logs, start/stop/restart. Live updates via `docker events` stream. Local sessions only.
- **Bookmarks** — Saved directory shortcuts with per-host namespacing for remote sessions.

Toggle panels via status bar icons or extend with [custom plugins](/plugin-development).

## Git Integration

The status bar shows the current git branch name. Click it to open a branch switcher showing both local and remote branches. The Git sidebar panel shows changed files with inline stage/unstage actions.

## Status Bar

Configurable segments displayed at the bottom of the window:

- **Environment** — Colored dot + label showing the active environment (`local`, `ssh`, `mosh`, `docker`, `kubectl`, and other supported remote types)
- **Git branch** — Current branch name with switcher popup
- **Path** — Current working directory
- **Running process** — Foreground process (zsh, node, vim, etc.)
- **Plugin icons** — Toggle icons for Files, Git, Docker, Bookmarks panels
- **Pane count** — Number of active panes and tabs
- **Clock** — Current time

Toggle individual segments in **Settings → Status Bar**, including path visibility and git branch display.

## Settings

Open with **Cmd+,**. Organized in tabs:

| Tab            | Options                                                                                           |
| -------------- | ------------------------------------------------------------------------------------------------- |
| **Theme**      | 32 color themes with live preview                                                                 |
| **Appearance** | Terminal font, font size, cursor style (block/beam/underline/outline); sidebar font and font size |
| **Status Bar** | Toggle individual segments (connection, path, git branch, process, pane count, clock)             |
| **Layout**     | Sidebar position (toggle with Cmd+B), workspace bar position, tab overflow mode                   |
| **Plugins**    | Enable/disable plugins, per-plugin settings (file explorer icons/hidden files, git settings)      |
| **Shortcuts**  | Keyboard shortcut reference                                                                       |
