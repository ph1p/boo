# Contributing

## Setup

1. Fork and clone the repository
2. `make setup` to build all dependencies
3. `make test` to verify everything works
4. Install swift-format: `brew install swift-format`

## Code Style

Boo uses [swift-format](https://github.com/swiftlang/swift-format) (config in `.swift-format`).

```bash
make format   # Format all source files
make lint     # Check for violations (CI runs this)
```

Rules: 4-space indent, 120-char lines, zero compiler warnings, all UI on main thread.

## Making Changes

1. Create a feature branch from `main` (e.g., `feature/plugin-name`, `fix/tab-close-crash`)
2. Make changes
3. `make format && make test && make lint`
4. Push and open a pull request

### Adding a Plugin

1. Create `Boo/Plugins/YourPlugin/YourPlugin.swift` conforming to `BooPluginProtocol`
2. Register in `PluginRegistry.registerBuiltins()`
3. Add tests in `Tests/BooTests/Plugins/YourPlugin/`

See [Plugin Development](/plugin-development) for the full guide.

## Testing

```bash
make test                              # Run all tests
swift test --filter PluginTests        # Run specific suite
```

## Release

Releases are automated via GitHub Actions using [semantic-release](https://github.com/semantic-release/semantic-release) with conventional commits.

```bash
make dist   # Manual: release build + sign + notarize + DMG
```

Requires: `SIGNING_IDENTITY`, `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_PASSWORD` env vars.

## Conventions

- `TerminalColor.cgColor`/`.nsColor` for color conversions
- `AppSettings.shared` with auto-notification for settings
- `RemoteExplorer.shellEscPath()` for remote paths
- `AppSettings.shared.pluginBool/pluginString/pluginDouble` for plugin settings
