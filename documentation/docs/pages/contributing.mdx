# Contributing

## Development Setup

1. Fork and clone the repository
2. Run `make setup` to build all dependencies
3. Run `make test` to verify everything works
4. Install `swift-format` for linting: `brew install swift-format`

## Code Style

Exterm uses [swift-format](https://github.com/swiftlang/swift-format) for consistent code formatting. Configuration is in `.swift-format` at the project root.

```bash
make format   # Format all source files in-place
make lint     # Check for style violations (fails on issues)
```

**Rules:**
- 4-space indentation
- 120 character line length
- Zero compiler warnings policy
- All UI operations on the main thread

CI runs `make lint` on every pull request. Format your code before pushing.

## Making Changes

### Branch Strategy

- `main` is the release branch
- Create feature branches from `main`
- Use descriptive branch names: `feature/plugin-name`, `fix/tab-close-crash`

### Pull Request Process

1. Create a feature branch
2. Make your changes
3. Run `make format` to format code
4. Run `make test` to verify all tests pass
5. Run `make lint` to check for style issues
6. Push and open a pull request
7. CI will automatically run lint, build, and tests

### Adding a Plugin

1. Create `Exterm/Plugins/YourPlugin/YourPlugin.swift`
2. Conform to `ExtermPluginProtocol`
3. Register in `PluginRegistry.registerBuiltins()`
4. Add tests in `Tests/ExtermTests/Plugins/YourPlugin/`
5. No `Package.swift` changes needed

See [Plugin Development](/plugin-development) for the full guide.

## Testing

```bash
make test             # Run all tests
swift test --filter PluginTests  # Run specific test suite
```

Tests cover models, plugins, services, terminal bridge, remote explorer, and SSH control. Aim for test coverage on any new functionality.

## Release Process

Releases are automated via GitHub Actions on tag push:

```bash
git tag v0.2.0
git push origin v0.2.0
```

This triggers the release workflow which builds, signs, notarizes, and publishes a DMG to GitHub Releases.

### Manual Release

```bash
make dist   # Full pipeline: release build, app bundle, sign, notarize, DMG
```

Requires environment variables for signing:
- `SIGNING_IDENTITY` — Developer ID certificate name
- `APPLE_ID` — Apple ID email
- `APPLE_TEAM_ID` — Developer Team ID
- `APPLE_PASSWORD` — App-specific password

## Conventions

- Use `TerminalColor.cgColor`/`.nsColor` extensions, not inline conversions
- Settings via `AppSettings.shared` with auto-notification
- Safe screen access via `screenRead`/`screenWrite` helpers
- Use `RemoteExplorer.shellEscPath()` for remote paths (preserves tilde expansion)
- Plugin settings: `AppSettings.shared.pluginBool/pluginString/pluginDouble`
